# Детекция занятости столика по видео
Скрипт определяет, пуст ли стол, к нему подошли или за ним сидят.
Основа — детекция людей через YOLO + вручную заданная зона столика.
## Быстрый старт
```bash
pip install -r requirements.txt
python main.py --video videos/video1.mp4
```

- На первом кадре откроется окно: **обведите зону места за столом многоугольником по кликам**. (В задании требовалось задавать четырехугольную рамку, но мне показалось это слишком неудобно и не универсально)
- ЛКМ: ставить вершины - Enter: завершить (нужно минимум 3 точки)
- Z: отменить последнюю точку
- ESC: выход
- Результат: **output.mp4** (цвет зоны: зелёный — empty, жёлтый — approach, красный — sat).
- Дополнительно: **events.csv**, **report.txt**, краткая статистика в консоли.

## Что считается событием
* empty — зона пустая (или эпизод закончился)
* approach — человек зашёл в зону, но быстро ушёл
* sat — человек находился в зоне достаточно долго (считаем, что сел)

## Опции
- --output other.mp4 — имя выходного файла
- --conf 0.35 — порог уверенности детекции человека
- --min_seat_time_sec 5.0 — если человек был в зоне >= этого времени, считаем что он **сел** (sat)
- --min_approach_time_sec 1.0 — минимальная длительность эпизода в зоне, чтобы считать это настоящим approach (а не шумом)
- --leave_grace_sec (по умолчанию 1.0) — сколько секунд можно пропустить детекцию из зоны перед завершением эпизода (полезно при перекрытиях/фликере)

## Пайплайн
1. Детекция
На каждый кадр запускается YOLO:
results = model.predict(frame, classes=[0])
Берём только людей (class=0).
2. Проверка попадания в зону
Зона — произвольный многоугольник.
Для каждого bbox считаются 3 точки:
центр
низ (ноги)
уровень бёдер
(cx, foot_y)
(cx, mid_y)
(cx, hip_y)
Если хотя бы одна точка внутри полигона, значит человек "в зоне".
Это нужно, потому что:
у сидящего могут не попадать ноги в зону
bbox часто гуляет
часть тела может быть перекрыта
3. Стабилизация
Детекция не идеальна, т.е. иногда человек перекрывается другими объектами сцены.
Чтобы не рвать эпизод:
```bash
effective_in_zone = raw_in_zone or (
            absence_start_time is not None and (time_sec - absence_start_time) < leave_grace_sec
        )
```
То есть допускаем возможные короткие пропуски.
4. Эпизоды
Система работает как простой state machine:
empty -> approach -> sat -> empty
Когда человек впервые появляется в зоне:
фиксируем episode_start_time
Дальше:
если пробыл >= min_approach_time_sec => approach
если пробыл >= min_seat_time_sec => sat
Когда человек исчезает=> эпизод закрывается (empty)
5. Логирование
События пишутся в формате csv:
frame,time_sec,event
123,4.92,approach
200,8.01,sat
350,14.2,empty
6. Метрики
После обработки считаются:
среднее время empty -> approach
среднее время empty -> sat
Это даёт представление о поведении людей у стола.

## Проблема

YOLO не понимает контекст.
Если официант долго стоит в зоне, то система думает, что он "сел".
***Почему так происходит:***
модель видит только "человек", не роль
логика завязана на время, а не на позу

## Возможности улучшения

* различать разных людей, а не просто факт присутствия
* Добавить позу (pose estimation): отличать "сидит" vs "стоит"
* Фильтрация по скорости движения: официант обычно движется, сидящий — нет
* Классификация роли (официант / гость): отдельная модель или эвристика
* Сделать возможность добавить две зоны, чтобы разметить только диваны для предотвращения миспредиктов официантов моделью.

## Вывод
Данная система может:
* Детектировать фактприсутствия человека в заданной области
* Отличать "Подошел" и "Сел"
* Игнорировать шумы
* Игнорировать перекрытия человека, занимающего стол
* Оставляет метку "Сел" даже если за столом несколько людей то тех пор, пока последний человек не покинет "зону"
